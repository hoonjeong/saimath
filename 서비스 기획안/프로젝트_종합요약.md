# 사이수학 입학테스트 시스템 - 프로젝트 요약

## 📋 프로젝트 개요

### 서비스 소개
초등학생을 대상으로 한 온라인 수학 입학 테스트 시스템입니다. 관리자(학원 선생님)가 학년별/학기별 시험지를 만들고, 학생은 간단한 정보 입력 후 25문제를 풀어 즉시 채점 결과를 확인할 수 있습니다.

### 핵심 가치
- **관리자**: 시험지 제작 및 학생 결과 분석을 통한 데이터 기반 상담
- **학생**: 간편한 온라인 테스트로 실력 진단 및 즉각적인 피드백
- **학부모**: 객관적인 데이터로 자녀의 실력 확인 및 신뢰도 향상

### 주요 기능
1. **관리자 페이지**
   - 시험지 생성 (학년/학기별, 25문제)
   - 학생 결과 조회 및 분석
   
2. **학생 페이지**
   - 정보 입력 (이름, 전화번호, 학년, 학기)
   - 25문제 온라인 풀이
   - 즉시 채점 및 결과 확인

## 🎯 타겟 사용자

### 주요 사용자 (Primary)
- **관리자**: 학원 선생님, 수학 강사
  - 연령: 25~50대
  - 목적: 입학 상담용 테스트 관리, 학생 실력 진단
  - 사용 빈도: 주 5~10회

- **학생**: 초등학생 (1~6학년)
  - 연령: 8~13세
  - 목적: 입학 전 실력 테스트
  - 사용 빈도: 1회 (재응시 가능)

### 간접 사용자 (Secondary)
- **학부모**: 자녀의 결과를 함께 확인
  - 역할: 학생 정보 입력 보조, 결과 확인

## 🔧 기술 요구사항

### 데이터 저장
- **로컬 데이터베이스** 사용
- 인터넷 연결 불필요
- 빠른 응답 속도 보장

### 데이터 구조

#### 1. 시험지 테이블 (TestPaper)
```
- id (PK)
- grade (1~6)
- semester (1 or 2)
- created_at (생성 일시)
- questions (25개 문제 JSON)
  [
    {
      question_number: 1,
      image_url: "...",  // optional
      choices: ["선지1", "선지2", "선지3", "선지4"],
      answer: 3  // 정답 번호 (1~4)
    },
    ...
  ]
```

#### 2. 응시 결과 테이블 (TestResult)
```
- id (PK)
- student_name (학생 이름)
- student_phone (전화번호, 암호화)
- grade (학년)
- semester (학기)
- test_paper_id (FK)
- taken_at (응시 일시)
- answers (25개 답안 JSON)
  [
    {
      question_number: 1,
      selected_answer: 3
    },
    ...
  ]
- score (총점)
- correct_count (정답 개수)
```

### 지원 환경
- **웹 브라우저**: Chrome, Safari, Edge (최신 버전)
- **디바이스**: PC, 태블릿, 모바일 (반응형)
- **해상도**: 320px ~ 1920px

## 🎨 UI/UX 원칙

### 디자인 철학
1. **단순성**: 불필요한 요소 제거, 핵심 기능에 집중
2. **직관성**: 설명 없이도 사용 가능한 인터페이스
3. **즉각성**: 모든 액션에 즉시 피드백 제공
4. **친근함**: 학생이 긴장하지 않도록 따뜻한 색상과 메시지

### 색상 전략
- **주 색상 (Primary)**: 파란색 계열 (#2196F3)
  - 신뢰감, 안정감
  - 버튼, 선택 상태
  
- **보조 색상 (Secondary)**: 초록색 계열 (#4CAF50)
  - 성공, 정답
  - 성공 메시지, 정답 표시
  
- **강조 색상 (Accent)**: 빨간색 계열 (#F44336)
  - 주의, 오답
  - 에러 메시지, 오답 표시

### 타이포그래피
- **제목**: 20~28px, Bold
- **본문**: 16~18px, Regular
- **버튼**: 16~18px, Medium
- **작은 텍스트**: 14px, Regular

### 모바일 최적화
- 터치 영역: 최소 44x44px
- 버튼 간격: 최소 8px
- 한 화면에 핵심 정보만 표시
- 스크롤 최소화

## 📱 주요 사용자 플로우

### 1. 관리자 - 시험지 제작
```
로그인 → 시험지 만들기 → 학년/학기 선택 → 25문제 입력 
→ 각 문제마다 (이미지 업로드 + 선지 4개 + 정답) 
→ 저장 → 완료
```
**소요 시간**: 약 30~60분

### 2. 학생 - 테스트 응시
```
시작 화면 → 정보 입력 (이름, 전화번호, 학년, 학기) 
→ 25문제 순차 풀이 (각 문제: 선지 선택 → 다음) 
→ 제출 → 즉시 채점 → 결과 확인
```
**소요 시간**: 약 20~40분

### 3. 관리자 - 결과 분석
```
로그인 → 결과 보기 → 학생 검색/선택 → 응시 이력 확인 
→ 특정 테스트 선택 → 문제별 정답/오답 확인 
→ 인쇄 또는 엑셀 다운로드
```
**소요 시간**: 약 5~15분

## ⚙️ 주요 비즈니스 규칙

### 시험지
- 1개 학년-학기 조합 = 1개 시험지
- 정확히 25문제로 구성
- 각 문제: 선지 4개(필수), 정답 1개(필수), 이미지(선택)
- 이미지: JPG/PNG, 최대 5MB

### 응시
- 정보 입력 필수: 이름, 전화번호, 학년, 학기
- 순차 진행: 1번 → 25번 (건너뛰기 불가, 뒤로가기 불가)
- 선지 선택 필수 (미선택 시 다음 버튼 비활성화)
- 중복 응시 허용 (동일 학생 여러 번 가능)

### 채점
- 자동 채점: 제출 즉시
- 각 문제 4점, 총 100점 만점
- 정답 = 4점, 오답 = 0점 (부분 점수 없음)

### 결과
- 관리자만 조회 가능
- 학생 식별: 이름 + 전화번호 조합
- 기본 정렬: 최근 응시일 내림차순
- 필터: 학년, 학기, 기간별

## 🚀 개발 우선순위

### Phase 1: MVP (Minimum Viable Product) - 4주
**목표**: 핵심 기능만으로 빠르게 출시

✅ **필수 기능**:
1. 관리자 로그인
2. 시험지 생성 (학년/학기 선택, 25문제 입력, 저장)
3. 학생 정보 입력
4. 25문제 풀이 (순차 진행)
5. 자동 채점 및 결과 표시
6. 결과 조회 (학생 목록, 상세 결과)

❌ **제외 기능**:
- 이미지 업로드 (텍스트만)
- 필터 및 검색 (전체 목록만)
- 인쇄/엑셀 다운로드
- 임시 저장
- 통계 차트

### Phase 2: Enhancement - 2주
**목표**: 사용성 개선 및 편의 기능 추가

✅ **추가 기능**:
1. 문제 이미지 업로드
2. 임시 저장 기능
3. 학생 검색 기능
4. 인쇄 기능
5. 에러 처리 강화

### Phase 3: Advanced - 2주
**목표**: 고급 기능 및 데이터 분석

✅ **추가 기능**:
1. 엑셀 다운로드
